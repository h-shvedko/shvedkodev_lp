# ADR-0001: Website Improvements Based on GA4 Analytics (90-Day Review)

- **Status:** Proposed
- **Date:** 2026-04-15
- **Property:** ShvedkoDev Website (GA4 `486908560`)
- **Period analysed:** 2026-01-15 – 2026-04-15 (90 days)

---

## Context

A full GA4 data pull was performed to surface concrete, evidence-based improvements for the ShvedkoDev landing page. The following dimensions were analysed: traffic by date, channel, source/medium, page, landing page, device category, and country.

## Decisions & Recommendations

### P1 — Critical

**ADR-0001-A: Implement bot/spam filtering**
- Bot traffic from China, Singapore, and unresolved sessions (~22% of total) is polluting all metrics.
- Action: Add a GA4 IP exclusion filter, enable "Define internal traffic" exclusions, and consider adding a referral exclusion list.
- *Why:* Real bounce rate for German traffic is likely 10–15pp lower than reported; paid campaign performance can't be judged accurately with spam inflating figures.

**ADR-0001-B: Reduce homepage bounce rate**
- Homepage bounce rate is 71.8% at 17s avg. Users leave before understanding the offer.
- Action: Strengthen the above-the-fold section with a clear value proposition, visible CTA ("View Portfolio" / "Get in Touch"), and a trust signal (e.g. client logos, years of experience).
- *Why:* 369 out of 533 sessions land here. Reducing bounce by even 15% means ~55 more engaged visitors per 90 days.

**ADR-0001-C: Fix /index.html duplicate content**
- GA4 shows `/index.html` as a separate URL with 3 sessions. This is a technical SEO issue.
- Action: Ensure the server (or `dist/` config) returns a 301 redirect from `/index.html` → `/`.
- Also observed: `/data-protection` (no trailing slash) and `/services` (no trailing slash) exist alongside their canonical `/slug/` versions.
- Action: Enforce consistent trailing-slash redirects across the board.

---

### P2 — High Priority

**ADR-0001-D: Invest in SEO content instead of (or before) Google CPC**
- Organic Search delivers 116s avg session, 27.8% bounce — by far the best channel quality.
- Google CPC delivers 81.7% bounce at 27s avg — users click and immediately leave.
- Action: Pause or drastically reduce CPC budget. Allocate that effort to on-page SEO: target 3–5 long-tail keywords relevant to the primary market (e.g. "Webentwickler Berlin", "freelance web developer Germany"), add meta descriptions, structured data (Person, LocalBusiness), and a basic blog/case-study section.

**ADR-0001-E: Elevate the Portfolio page in the user journey**
- `/portfolio/` has the lowest bounce rate (18%) and second-longest avg session (26s) of all pages — visitors who get there are highly engaged.
- Current data shows only 52 views vs. 403 homepage views — very few users navigate to it.
- Action: Add a prominent "See my work" CTA in the hero section and consider surfacing 2–3 portfolio thumbnails directly on the homepage.

**ADR-0001-F: Add conversion tracking beyond homepage**
- 46 conversions recorded, all attributed to homepage/direct. Contact page, services page, and portfolio page show 0 conversions despite meaningful engagement.
- Action: Track `contact_form_submit` (Formspree success response), `click_email`, and `click_phone` as GA4 events across all pages. This is a configuration gap, not a behaviour gap.

---

### P3 — Improvements

**ADR-0001-G: Leverage LinkedIn more actively**
- LinkedIn referral produces 9 sessions with 44% bounce and 65s avg — much better quality than any paid channel.
- Action: Post regular updates (case studies, project snippets) on LinkedIn linking to specific portfolio items or the services page. No ad spend required.

**ADR-0001-H: Improve non-homepage landing page experience**
- `/about`, `/contact`, `/services`, and `/portfolio` all have 94–100% bounce when used as landing pages.
- Action: Each of these pages needs a header section that orients a first-time visitor (brief intro + nav breadcrumb) and at least one clear next-step CTA so direct links from LinkedIn/social posts don't dead-end.

**ADR-0001-I: Investigate Austrian & DACH-region traffic**
- Austria (15 sessions, 86.7% bounce) is a logical expansion market (German-speaking, EU). Currently high bounce.
- Action: Ensure copy and case studies resonate with DACH context; add Austrian and Swiss city-specific landing pages if targeting expands.

**ADR-0001-J: Consider a Ukrainian/Eastern European page variant**
- Ukraine (8 sessions, 12.5% bounce) and Russia (13 sessions, 7.7% bounce) are highly engaged but likely personal-network visits.
- Action: If these represent referral pipeline, a Ukrainian-language or bilingual variant (`/uk/`) could formalise that audience.

### ADR-0001-L: Static Blog System

**Context**

Organic Search is the highest-quality traffic channel (27.8% bounce, 116s avg session) yet produces only 19 sessions in 90 days. The fastest way to grow organic traffic without a server is a static blog — HTML files authored locally, committed to git, and deployed via the existing pipeline.

**Architecture Decision**

Keep the blog entirely within the existing Gulp + rigger static pipeline. No SSG (Jekyll, Hugo, etc.) is introduced — it adds a dependency and a different mental model. Instead:

```
src/
  html/
    blog/
      index.html           ← Blog index (list of all posts)
      _post-template.html  ← Rigger template (shared layout for all posts)
      posts/
        2026-04-craftly-ai-platform.html
        2026-03-laravel-12-livewire.html
        ...
```

**Workflow for a new blog post**

1. Copy `src/html/blog/_post-template.html` → `src/html/blog/posts/YYYY-MM-slug.html`
2. Fill in front-matter comment block at the top (title, date, description, tags)
3. Write the post body in the HTML file
4. Manually add the post card to `src/html/blog/index.html`
5. `gulp build` or `gulp dist` → HTML files appear in `dist/blog/posts/`
6. Commit and push to `main` → GitHub Actions deploys to hosting (see ADR-0001-M)

**Post template structure**

```html
<!--
  title:       Laravel 12 What's New
  date:        2026-03-10
  description: A practical overview of Laravel 12 features for PHP developers.
  tags:        laravel, php, backend
-->
//= ../layout/head.html
//= ../layout/header.html
<main class="blog-post">
  <article>
    <h1><!-- post title --></h1>
    <time datetime="YYYY-MM-DD"><!-- human date --></time>
    <!-- post content -->
  </article>
</main>
//= ../layout/footer.html
```

**Blog index page**

`src/html/blog/index.html` — a hand-maintained list of post cards (title, date, excerpt, read-more link). No JavaScript pagination needed until there are 20+ posts.

**SEO per post**

Each post file includes via `//= head.html` the standard meta block, but the post-specific `<title>`, `<meta name="description">`, and canonical URL are injected as override variables at the top of the file before the rigger include.

**Canonical URLs** — all posts at `/blog/posts/slug/` (trailing slash enforced via server redirect rule or `.htaccess`).

**Internal linking** — the blog index is linked from the main nav and from a "Latest Articles" section on the homepage (2–3 most recent post titles, updated manually when new posts are published).

**Suggested first posts** (derived from LinkedIn activity and Craftly launch):

| Slug | Topic | Target keyword |
|---|---|---|
| `craftly-ki-plattform-laravel` | Craftly launch story, what it does, why PHP | "KI Laravel App Generator" |

### ADR-0001-M: FTP Deployment via GitHub Actions

**Context**

The current deploy workflow is manual (local `gulp dist` → FTP client). Every deploy is a friction point and a potential inconsistency. Automating deployment on push to `main` removes the manual step, makes CI state the single source of truth, and enables future collaborators to deploy without needing FTP credentials locally.

**Decision**

Use a GitHub Actions workflow that:
1. Triggers on `push` to `main`
2. Runs `npm ci && gulp dist` to produce a clean `dist/`
3. Uploads `dist/` to the hosting root via FTP using the `SamKirkland/FTP-Deploy-Action` action

**Workflow file: `.github/workflows/deploy.yml`**

```yaml
name: Build & Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build production assets
        run: npx gulp dist

      - name: Deploy via FTP
        uses: SamKirkland/FTP-Deploy-Action@v4.3.5
        with:
          server: ${{ secrets.FTP_SERVER }}
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          local-dir: ./dist/
          server-dir: ${{ secrets.FTP_REMOTE_DIR }}
          exclude: |
            **/.git*
            **/.git*/**
            **/node_modules/**
```

**GitHub Secrets required**

| Secret name | Value |
|---|---|
| `FTP_SERVER` | Hosting FTP hostname (e.g. `ftp.shvedko.dev`) |
| `FTP_USERNAME` | FTP account username |
| `FTP_PASSWORD` | FTP account password |
| `FTP_REMOTE_DIR` | Remote root path (e.g. `/html/` or `./`) |

Add these in: GitHub repo → Settings → Secrets and variables → Actions → New repository secret.

**FTP-Deploy-Action behaviour**

- Uses `--delete` mode by default — remote files not present in `dist/` are removed. Verify this is safe (no server-side uploads or CMS-managed files in the web root).
- Uploads only changed files on subsequent runs (diff-based via `.ftp-deploy-sync-state.json` stored in the repo root — add this to `.gitignore`).

**`.gitignore` additions**

```
.ftp-deploy-sync-state.json
```

**Manual deploy fallback**

For emergency hotfixes that cannot wait for CI: `gulp dist` locally, then upload `dist/` manually via FTP client as before.

### ADR-0001-N: Preserve `dist/craftly/` in `src/`

**Context**

`dist/craftly/` contains a standalone Craftly landing page (German-language, Tailwind CSS, vanilla JS). It exists only in `dist/` — any `gulp clean:build` or `gulp dist` run will wipe it. This is a production page that must survive rebuilds.

**Current state of `dist/craftly/`**

```
dist/craftly/
  index.html          ← German LP: "Webanwendungen in 5 Minuten statt Wochen"
  index-en.html       ← English variant
  assets/
    img/
      screenshot1–4.png
      placeholder-*.svg
      _- visual selection.svg
```

The page uses Tailwind CSS via CDN (`cdn.tailwindcss.com`) and Google Fonts — no build pipeline dependency. It is self-contained.

**Decision**

Move the Craftly landing page out of `dist/` and into `src/` as a static passthrough directory:

```
src/
  craftly/
    index.html
    index-en.html
    assets/
      img/  ← screenshots and placeholders
```

**Gulp integration**

Add a `craftly:build` task to `gulpfile.js` that copies `src/craftly/**/*` verbatim to `dist/craftly/`:

```js
// gulpfile.js — add to task definitions
gulp.task('craftly:build', function () {
  return gulp.src(path.src.craftly, { encoding: false })
    .pipe(gulp.dest(path.build.craftly));
});
```

Add path entries to the `path` config object:

```js
path.src.craftly   = 'src/craftly/**/*';
path.build.craftly = 'dist/craftly/';
```

Include `craftly:build` in both `gulp.series` calls for `build` and `dist` tasks, and add `src/craftly/**/*` to the `gulp.watch` watcher.

**Pre-deploy fix**

The `index.html` currently has a TODO comment:
```html
<!-- TODO before deploy: change to absolute URL e.g. https://shvedko.dev/craftly/assets/img/screenshot1.png -->
<meta property="og:image" content="./assets/img/screenshot1.png">
```
Update OG image URLs to absolute `https://shvedko.dev/craftly/assets/img/screenshot1.png` before moving to `src/`.

### ADR-0001-O: Craftly SEO Visibility — Homepage Banner + Portfolio Card

**Context**

Craftly is the flagship product — "the world's first AI agent platform for Laravel and WordPress". It has its own landing page at `/craftly/` but zero cross-linking from the main site. The homepage and portfolio page have no mention of it. This is a missed opportunity both for SEO (internal links) and for lead generation (visitors who would be interested in Craftly never see it).

**GA4 signal:** `/craftly/` does not appear in the page report at all — it has received zero tracked visits in 90 days. The page is essentially invisible.

**Decision A — Homepage banner / feature card**

Add a visually prominent card or banner in the homepage's main content area (below the hero, above or within the services section) promoting Craftly:

```
┌─────────────────────────────────────────────────────┐
│  ✦ Craftly                              [NEW]        │
```

- Links to `/craftly/` (internal) — passes SEO equity

- Mobile-friendly — collapses to full-width card
