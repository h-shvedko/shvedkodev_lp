# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Layout

```
src/                 # Source files (edit these)
├── html/            # HTML pages + layout/ + components/ partials
├── scss/            # main.scss imports base/, components/, layout/, vendors/
├── js/              # Top-level *.js are entry points; nested files are includes
├── sourceimages/    # Images copied to dist/images/
└── fonts/
dist/                # Build output — do not edit
gulpfile.js
```

## Commands

- `npm install` — install dependencies
- `gulp` (default) — clean, build, start BrowserSync on `localhost:9001`, watch
- `gulp build` — dev build (with sourcemaps)
- `gulp dist` — production build (minified, no sourcemaps)
- `gulp watch` — watch only
- `gulp webserver` — BrowserSync only, serving `./dist`
- Individual tasks: `html:build`, `css:build` / `css:dist`, `js:build` / `js:dist`, `image:build`, `fonts:build`, `clean:build`, `cache:clear`

There is no test suite, linter, or typechecker configured.

## Build Pipeline (gulpfile.js)

- **HTML**: `src/html/**/*.html` → processed by [gulp-rigger](https://www.npmjs.com/package/gulp-rigger), which resolves `//= path/to/file.html` include directives. Pages live in `src/html/` (e.g. `index.html`, `services/`, `about/`); shared chunks live in `src/html/layout/` (`head.html`, `header.html`, `footer.html`, `google-analytics.html`) and `src/html/components/`. Output goes flat into `dist/`.
  - Note: the `path.src.html` config (`src/*.html`) is unused — the actual `html:build` task hardcodes `src/html/**/*.html`.
- **CSS**: only `src/scss/main.scss` is compiled (via `gulp-sass` + `sass`); it `@import`s everything else. Autoprefixer uses the `browserslist` field in `package.json`. Both `main.css` and `main.min.css` are emitted to `dist/css/`.
- **JS**: only top-level files matching `src/js/*.js` are compiled — also via `gulp-rigger`, so use `//= ./subdir/file.js` directives to pull in nested modules. Both unminified and `.min.js` are emitted.
- **Images**: `src/sourceimages/**/*` is copied straight to `dist/images/` — imagemin/mozjpeg/pngquant are required at the top of the gulpfile but not actually wired into the pipeline. Optimize images before committing if needed.
- **Fonts**: copied as-is from `src/fonts/` to `dist/fonts/`.
- `src/robots.txt` and `src/sitemap.xml` are not part of any gulp task — copy manually if changed.

## Conventions

- Edit only `src/`. The `dist/` folder is regenerated by `gulp build` / `gulp dist`.
- When adding a new HTML page, place it under `src/html/` and reference shared layout/components through rigger `//=` includes (see `src/html/index.html` for the pattern).
- When adding new SCSS partials, import them from `src/scss/main.scss` (or from a partial that is already imported) — standalone `.scss` files are not compiled.
- When adding new JS modules, either add an entry file directly under `src/js/` or include it from an existing entry via `//=`.
